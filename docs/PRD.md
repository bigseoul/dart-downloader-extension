# DART 다운로더 For AI — Product Requirements Document (PRD)

> **Version** 1.0.3  
> **Last Updated** 2026-03-26  
> **Status** Current

---

## 1. 개요

**DART 다운로더 For AI**는 금융감독원 전자공시시스템(DART) 문서 열람 페이지에서 동작하는 Chrome 확장 프로그램이다. 사용자는 문서 목차를 트리 형태로 확인하고 원하는 범위를 선택해, AI 입력에 적합한 **구조화 텍스트**로 저장할 수 있다.

현재 제품은 두 가지 문서 유형을 지원한다.

- **목차형 문서**: `#listTree`의 `jstree`를 읽어 트리 UI를 렌더링한다.
- **단일 페이지 문서**: 목차를 렌더링하지 않고 **전체 문서 저장** UI를 즉시 표시한다.

---

## 2. 문제 정의

DART 문서는 뷰어 내에서 읽기에는 적합하지만, AI 입력용으로 재사용하기에는 다음 제약이 있다.

- 필요한 섹션만 골라 구조적으로 저장하기 어렵다.
- 문서마다 목차형/단일 페이지형 구조가 달라 일관된 추출 흐름이 필요하다.
- 원본 HTML 그대로보다, 제목과 표 구조가 살아 있는 텍스트 포맷이 AI 입력에 더 적합하다.

---

## 3. 제품 목표

- 사용자가 DART 문서의 구조를 **빠르게 파악**할 수 있어야 한다.
- 선택한 범위를 AI 입력용 **구조화 텍스트**로 저장할 수 있어야 한다.
- 목차형 문서와 단일 페이지 문서를 **한 확장 UX 안에서 일관되게 처리**해야 한다.
- 단일 페이지 문서에서는 팝업 진입 시 불필요한 HTML 선fetch 없이 **저장 UI를 빠르게 노출**해야 한다.

---

## 4. 비목표

- DART 외 사이트 지원
- PDF, Excel 등 다른 포맷 변환
- 문서 요약, 질의응답, 벡터화 등 AI 후처리 기능
- 사용자 계정, 서버 저장, 클라우드 동기화

## 6. 사용자 경험

### 6.1 목차형 문서

1. 사용자가 DART 문서 열람 페이지를 연다.
2. 확장 팝업을 열면 목차 트리가 표시된다.
3. 사용자는 체크박스로 필요한 범위를 선택한다.
4. 저장 방식을 고른다.
   - `합쳐 저장`: 선택 내용을 하나의 `.txt`로 저장
   - `따로 저장`: 선택 내용을 항목별 `.txt`로 나눠 `.zip`으로 저장
5. 저장을 실행하면 진행 상태가 표시되고, 완료 후 브라우저 다운로드가 시작된다.

### 6.2 단일 페이지 문서

1. 사용자가 DART 단일 페이지 문서를 연다.
2. 확장 팝업은 트리 대신 **전체 문서 저장** 안내를 표시한다.
3. 사용자가 `전체 저장`을 누르면 그 시점에만 현재 문서 HTML을 가져와 구조화 텍스트로 변환한다.
4. 결과는 하나의 `.txt` 파일로 저장된다.

## 7. 핵심 기능 요구사항

### FR-1. DART 문서 판별

- 확장은 허용된 DART 호스트에서만 동작해야 한다.
- 비DART 페이지에서는 사용자에게 오류 메시지를 표시해야 한다.

### FR-2. 목차 추출

- content script는 페이지 컨텍스트에 `inject.js`를 주입해 `jstree` 데이터를 읽어야 한다.
- `jstree` 루트의 자식 노드를 재귀적으로 읽어 트리 JSON으로 반환해야 한다.
- 각 노드는 다음 메타데이터를 포함해야 한다.
  - `id`, `text`, `rcpNo`, `dcmNo`, `eleId`, `offset`, `length`, `dtd`, `children`

### FR-3. 단일 페이지 문서 지원

- `jstree`가 없더라도 `rcpNo/dcmNo`가 있으면 단일 페이지 문서로 판정할 수 있어야 한다.
- `#listTree`가 존재하더라도 실제 목차 수가 0인 경우(`cnt == 0`, empty tree)도 단일 페이지로 판정해야 한다.
- 단일 페이지 문서에서는 트리 렌더링이나 전체 HTML 선fetch 없이 **즉시 저장 UI**를 보여줘야 한다.
- 실제 HTML 직렬화는 사용자가 저장 버튼을 눌렀을 때만 수행해야 한다.

### FR-4. 트리 UI

- 팝업은 계층 구조 트리를 렌더링해야 한다.
- 각 노드는 체크박스를 가져야 한다.
- 자식이 있는 노드는 펼치기/접기 토글을 제공해야 한다.
- `전체 선택`, `선택 해제`, `모두 펼치기`, `모두 접기`를 제공해야 한다.
- 일부 자식만 선택된 부모는 `indeterminate` 상태를 표시해야 한다.

### FR-5. 저장 방식

- 저장 방식은 다음 두 가지를 제공해야 한다.
  - `합쳐 저장`: 선택된 항목들을 하나의 AI 입력용 텍스트로 합친다.
  - `따로 저장`: 선택된 항목들을 각각 AI 입력용 텍스트로 만든 뒤 ZIP으로 묶는다.
- 단일 페이지 문서에서는 `합쳐 저장`만 허용한다.

### FR-6. 중복 제거와 저장 단위 결정

- 부모 노드가 선택된 경우 하위 자식 노드는 중복 저장 대상에서 제외해야 한다.
- `합쳐 저장`에서는 유효 선택 노드를 그대로 저장 단위로 사용한다.
- `따로 저장`에서는 선택된 부모 노드를 리프 노드까지 확장해 항목별 파일을 생성한다.

### FR-7. 콘텐츠 가져오기

- 목차형 문서의 각 노드는 `viewer.do`를 fetch해 HTML을 가져와야 한다.
- 단일 페이지 문서는 현재 문서 또는 iframe 내용을 직렬화해야 한다.
- fetch에는 타임아웃이 적용되어야 한다.
  - 일반 노드: 15초
  - 단일 페이지 전체 문서: 30초

### FR-8. 구조화 텍스트 변환

- 원본 HTML은 AI 입력에 적합한 텍스트로 변환해야 한다.
- 변환 결과는 최소한 다음 정보를 유지해야 한다.
  - 문서 제목(`DOCUMENT:`)
  - 섹션(`SECTION:`)
  - 표 머리/행 정보(`HEADER:`, `ROW:` 등)
- `script`, `style`, `iframe` 등 노이즈 태그는 제거해야 한다.

### FR-9. 파일명 규칙

- 기본 파일명 접두사는 `기수_회사명_문서유형` 조합을 사용한다.
- 메타데이터가 없으면 `dart_documents`를 fallback으로 사용한다.
- OS 금지 문자는 `_`로 치환해야 한다.
- 출력 파일 규칙:
  - `합쳐 저장`: `{baseName}.txt`
  - `따로 저장`: `{baseName}.zip`
  - ZIP 내부: `{baseName}_{항목명}.txt`

### FR-10. 문서 메타데이터 추출

- 회사명은 기본적으로 페이지 타이틀에서 추출해야 한다.
- 문서 유형은 현재 `dcmNo`와 매칭되는 select option 텍스트를 우선 사용해야 한다.
- 기수(`제 n 기`)는 현재 표시 문서 본문, 보고서 후보 노드, fallback fetch 순으로 추출해야 한다.
- 단일 페이지 문서에서는 메타데이터 조회를 백그라운드로 수행해야 하며, UI 표시를 블로킹하지 않아야 한다.

### FR-11. 진행 상태 및 실패 처리

- 저장 중 진행 오버레이와 프로그레스 바를 표시해야 한다.
- 전체 실패 시 사용자에게 실패 요약을 보여줘야 한다.
- 일부 실패 시 나머지만 저장할지 사용자에게 확인받아야 한다.

### FR-12. 진단 로그

- 성능 병목 분석을 위해 popup/content/inject 각 단계에서 콘솔 타이밍 로그를 남겨야 한다.
- 주요 로그 포인트 예:
  - `[DART AI][popup] tabs.query.done`
  - `[DART AI][popup] getTreeData.done`
  - `[DART AI][popup] singlePage.ui.ready`
  - `[DART AI][content] inject.start`
  - `[DART AI][content] inject.result`
  - `[DART AI][content] getTreeData.done`
  - `[DART AI][inject] singlePage.fast-path`

## 8. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 성능 | 목차형 문서는 팝업 오픈 후 가능한 한 빠르게 트리를 표시해야 한다 |
| 성능 | 단일 페이지 문서는 HTML 선fetch 없이 저장 UI를 우선 보여줘야 한다 |
| 안정성 | DART 문서 비정상 상태, fetch 실패, 타임아웃을 사용자 메시지로 처리해야 한다 |

| 사용성 | 저장 방식과 현재 상태를 팝업에서 명확히 구분해야 한다 |
| 다운로드 | 브라우저 다운로드 API를 사용하고 `saveAs: true`로 저장 위치를 선택할 수 있어야 한다 |

## 9. 시스템 아키텍처

```
┌────────────────────────────────────────────────────────────┐
│ Chrome Extension (Manifest V3)                            │

│   · 선택 상태 관리                                           │
│   · 구조화 텍스트 생성/ZIP 생성                               │
│   · 다운로드 실행                                             │
│                                                            │
│          chrome.tabs.sendMessage                           │
│                     │                                      │
│                     ▼                                      │
│  content.js                                                │
│   · inject.js 주입                                          │
│   · viewer.do fetch                                         │
│   · 메타데이터 추출                                           │

│   · 페이지 컨텍스트에서 jstree 접근                           │

```

### 9.1 주요 메시지

| 발신 → 수신 | 액션 | 목적 |
|---|---|---|
| popup → content | `getTreeData` | 목차형/단일 페이지 판별 및 트리 데이터 조회 |
| popup → content | `getDocMeta` | 회사명, 기수, 문서유형 조회 |
| popup → content | `fetchNodeHTML` | 개별 노드 HTML fetch |
| popup → content | `fetchSinglePageHTML` | 단일 페이지 전체 문서 직렬화 |
| inject → content | `DART_TREE_DATA` | 페이지 컨텍스트 추출 결과 전달 |

## 10. 오류 및 예외 처리

| 상황 | 처리 방식 |
|---|---|
| 비허용 호스트에서 실행 | 오류 메시지 표시 |
| 문서 자체가 비정상 상태 | 오류 코드와 메시지 반환 |
| `jstree` 로드 지연 | 짧은 재시도 후 실패 처리 |
| 빈 목차 단일 문서 | single-page fast path로 즉시 전환 |
| 개별 항목 fetch 실패 | 실패 목록에 누적 후 나머지 계속 진행 |
| 전체 항목 실패 | 전체 실패 메시지 표시 후 중단 |
| 일부 항목 실패 | 남은 항목만 저장할지 확인 |
| 선택 항목 없음 | 저장 버튼 비활성화 |

## 11. 현재 버전 기준 확인 포인트

- 확장 버전은 `1.0.3`이다.
- 단일 페이지 문서에서는 팝업 로딩 시 전체 HTML을 미리 가져오지 않는다.
- 단일 페이지 문서에서는 `buildSinglePageTree()` 기반 섹션 트리 렌더링을 사용하지 않는다.
- 단일 페이지 문서는 `전체 문서` 1개를 저장 대상으로 취급한다.
- `singlePage.fast-path`는 다음 두 케이스를 포함한다.
  - `#listTree` 자체가 없는 경우
  - `#listTree`는 있으나 실제 목차가 비어 있는 경우(`cnt == 0`, empty tree)
