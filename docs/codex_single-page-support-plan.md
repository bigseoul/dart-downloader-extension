# 단일 페이지형 DART 문서 대응 계획

## Summary

`dsaf001/main.do` 문서 중 `jstree` 사이드바가 없는 단일 페이지형 문서도 지원한다. 기존 `jstree` 추출을 우선 시도하고, 실패하면 현재 문서 DOM에서 섹션 구조를 합성해 팝업 트리로 제공한다.

단락 구분은 `중간 단위`를 기본으로 한다. 즉, 재무제표 본문(`재무상태표`, `손익계산서`, `현금흐름표` 등)은 독립 섹션으로 분리하고, 주석 구간은 `section-2`, `table-group-xbrl`, `H1-H6`, `2.1` 수준 번호형 제목을 기준으로 나눈다. `(1)`, `가.` 수준은 기본 다운로드 단위로 올리지 않는다. 단일 페이지형에서는 항상 `전체 문서` 항목도 함께 제공한다.

## Key Changes

### `content.js`

- `getTreeData`에서 기존 `jstree` 추출 실패 시 DOM 기반 fallback 수행
- fallback 노드는 기존 트리 포맷을 최대한 유지하되 `inline` 타입 식별값 추가
- `fetchNodeHTML`은 기존 `viewer.do` 노드와 DOM 분할 노드를 모두 처리

### DOM 기반 섹션 분리 규칙

- 최우선 경계: `p.section-2`, `p.table-group-xbrl`, `H1-H6`
- 추가 경계: 재무제표 제목(`재무상태표`, `손익계산서`, `포괄손익계산서`, `자본변동표`, `현금흐름표`)과 명확한 번호형 제목 `^\d+\.\d+\s+\S`
- 제외 경계: `(1)`, `가.`, `①` 등 세부 항목
- 각 섹션 범위는 "현재 경계 노드부터 다음 동급/상위 경계 직전까지"
- 문서 최상단에 `전체 문서` 가상 노드 추가

### 팝업 트리 동작

- 기존 체크박스 트리 UI 유지
- 단일 페이지형에서도 동일한 부모/자식 선택과 중복 제거 로직 사용
- `전체 문서` 선택 시 하위 분할 항목은 중복 다운로드에서 제외
- 파일명은 섹션 제목 기반, 전체 문서는 `{prefix}_전체 문서`

## Test Plan

- 기존 `jstree` 문서에서 기존 동작이 깨지지 않는지 확인
- 단일 페이지형 문서에서 팝업이 에러 대신 합성 트리를 표시하는지 확인
- 재무제표 본문이 `재무상태표`, `손익계산서` 등 독립 섹션으로 분리되는지 확인
- `재무제표 주석` 아래에 `1.`, `2.`, `2.1` 수준이 중간 단위로 나뉘는지 확인
- `전체 문서`만 다운로드, 개별 섹션만 다운로드, 부모/자식 동시 선택 시 중복 제거를 확인
- 제목 중복, 빈 섹션, 표만 있는 섹션에서 오분리 여부 확인

## Assumptions

- 단일 페이지형 문서에서도 주요 제목은 `section-2`, `table-group-xbrl`, 제목성 문단, 또는 `H1-H6`로 식별 가능하다
- 초반 재무제표 본문은 주석과 별개로 분리하는 것이 사용자 가치가 높다
- `(1)`, `가.` 수준 세분화는 과도하므로 이번 범위에서 제외한다
- 자동 분리가 완벽하지 않아도 `전체 문서` fallback이 있으면 사용성이 유지된다
